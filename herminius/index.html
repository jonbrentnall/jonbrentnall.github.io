Herminius

This client came to us to refresh their current website, it was previously a one-page static site that simply opened onto a full-screen image, with heavy images and unoptimized code. As it was a static site the client was unable to update any of the content themselves, which was a requirement for the new site.

herminius.com

The old site is still available via Wayback Machine

From the outset the site was built with speed and performance in mind, keeping the page weight and requests as low as possible. All the images are served up via srcset and the website attempts to use .webp if the browser supports it (falls back to png / jpg where applicable).

As budget constraints are always against us we weren't able to include some nice to have features that would have improved the usability of the site, such as preloading an internal page when the user hovers over the page link or lazy loading images. However, with that being said most pages still fully load in ~500ms.
